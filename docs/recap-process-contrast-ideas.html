Six cycles, then four, then five. Stop-and-wait to async is a third faster - and when async finishes, stop-and-wait still has a whole review to go. Async to worktrees gives a cycle back. Parallel building collapses the agent's time, but the reviewer reads the same things in the same order, so review stays the bottleneck. More agents just buys more idle agents and a merge.
