The gain is visible in time, not boxes.

The old loop serialises work and review. recap turns the same events into overlapping lanes, with feedback returning as the next ordered action.

Without recap: review blocks the line

The agent finishes a unit, then waits for attention. By the time feedback arrives, the work has to be reconstructed from chat, TODOs, and memory.

With recap: work and review overlap

The agent records proof and moves on. The reviewer inspects when ready. Feedback re-enters through `recap next` before fresh TODO work.

Idle time becomes useful time The agent starts the next item while the reviewer reviews the last one.
Feedback has a route back Requested changes return as priority work rather than interrupting from the side.
Review starts with evidence Each item carries the success criterion, recheck command, result, summary, and sha.
The queue owns priority `recap next` prevents channel-sweeping and keeps one clear next action in flight.
